ActionCard

ActionCardComponent es un componente reutilizable que representa una tarjeta interactiva con ícono, título y subtítulo. Está diseñada para funcionar como botón de navegación dentro de la aplicación, permitiendo redirección a una ruta específica al hacer clic.

Ejemplo

Inputs

  • ariaLabelKey: string — Clave de traducción para accesibilidad (atributo aria-label).
  • icon: string — Nombre del ícono SVG registrado que se mostrará en la tarjeta.
  • titleKey: string — Clave de traducción para el título principal de la tarjeta.
  • subtitleKey: string — Clave de traducción para el subtítulo descriptivo.
  • routerPath: string — Ruta a la que se navega al hacer clic en la tarjeta.

Comportamiento

Al hacer clic en la tarjeta, se ejecuta el método navigateTo(), que redirige al usuario mediante el Router de Angular. El componente está optimizado para accesibilidad y traducción dinámica usando translate.

PageHeaderComponent

PageHeaderComponent es un componente de encabezado reutilizable que se muestra en la parte superior de cada vista. Incluye el título de la página, un botón opcional de "volver", y un bloque de información del usuario con acceso rápido a configuración regional y cierre de sesión.

Ejemplo

Inputs

  • pageTitle: string — Clave de traducción para el título de la página.
  • backRoute: string — Ruta a la que se navega al hacer clic en el botón de "volver". Si no se proporciona, el botón no se muestra.

Contenido interno

  • Botón de volver: Se muestra solo si backRoute está definido. Usa mat-icon-button con el ícono arrow_back.
  • Título de la página: Traducido dinámicamente con translate.
  • Bloque de usuario: Muestra nombre, rol y un botón flotante con ícono de usuario.
  • Menú desplegable: Aparece al hacer clic en el botón de usuario. Incluye nombre, rol y opción de cierre de sesión.

Accesibilidad

Todos los elementos clave incluyen atributos aria-label traducibles para compatibilidad con lectores de pantalla. El botón de volver usa aria-label="Back" y el menú desplegable se describe con aria-label="userMenuLabel".

Comportamiento

  • goBack(): Navega a la ruta definida en backRoute.
  • toggleMenu(): Muestra u oculta el menú de usuario.
  • logout(): Ejecuta la lógica de cierre de sesión.

Botón de acción (mat-flat-button)

Este botón representa una acción principal dentro de una vista, como guardar configuraciones. Utiliza mat-flat-button de Angular Material con estilo plano, color primario, y soporte para traducción dinámica del texto. También puede deshabilitarse mediante una condición lógica.

Ejemplo

Ejemplo con estado deshabilitado

Propiedades

  • mat-flat-button: Aplica estilo plano sin sombra ni elevación.
  • color="primary": Usa el color primario del tema.
  • (click)="method()": Ejecuta una acción al hacer clic.
  • [disabled]: Desactiva el botón.
  • {{ 'saveButton' | translate }}: Traduce dinámicamente el texto del botón.

Comportamiento

El botón ejecuta el método method() definido en el componente. Puede deshabilitarse para evitar múltiples envíos o mientras se realiza una operación.

Botón plano con ícono (mat-button)

Este botón utiliza mat-button de Angular Material, con estilo plano y fondo transparente. Incluye un ícono SVG decorativo y texto traducible. Es ideal para acciones secundarias como descargar, compartir o abrir enlaces.

Ejemplo

Características

  • mat-button: Botón plano sin fondo ni sombra.
  • Ícono SVG: Se muestra antes del texto, decorativo.
  • Texto traducible: Compatible con translate.
  • Estado deshabilitado: Controlado con [disabled].

StatusMessageComponent (uso en mensajes de estado)

Este componente se utiliza para mostrar mensajes de éxito o error en la interfaz. Puede integrarse de forma embebida en el layout o mostrarse como mensaje flotante (toast) en la esquina inferior centrada. A continuación se muestran ejemplos de uso y los atributos disponibles.

Estado: mensaje exitoso embebido

Estado: mensaje de error flotante

Estado: mensaje temporal (desaparece automáticamente)

Inputs disponibles

  • type: Define el tipo de mensaje. Puede ser 'success' o 'error'.
  • messageKey: Clave de traducción para el contenido del mensaje.
  • float: Si es true, el mensaje se muestra como toast flotante centrado. Si es false (por defecto), se muestra embebido.
  • duration: Tiempo en milisegundos que el mensaje permanece visible. Si no se define, el mensaje queda estático.

Estados visuales esperados

  • Success: fondo #e6f4ea, texto #09A794, ícono check al final.
  • Error: fondo #fbe9e7, texto #EB5757, ícono close al final.

Accesibilidad

  • Usa role="status" para éxito y role="alert" para error.
  • Incluye aria-live para que el mensaje sea anunciado automáticamente por lectores de pantalla.
  • Los íconos están marcados como aria-hidden="true" para evitar redundancia.
